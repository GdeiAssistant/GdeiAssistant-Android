# 公开文案维护指南

## 适用范围

本指南用于维护 Android 客户端的 README、关于页、协议页、隐私说明、校园凭证说明、权限说明以及其他面向用户展示的公开文案。公开文案应写给真实用户阅读，不应出现内部草稿、实现说明或 AI 生成提示。

## 应当说明的内容

- 用户权利、用户义务和功能限制。
- 隐私处理、敏感信息处理和账户安全提示。
- 校园账号凭证的授权、使用范围、撤回授权、删除申请和快速认证控制。
- Android 权限用途、第三方服务使用范围和必要风险提示。
- 二手、跑腿/快递、表白墙、失物招领等用户内容和交易场景的风险说明。
- 项目不是学校官方服务、也不代表学校授权服务的说明。
- 投诉、举报、申诉和帮助反馈入口。

## 不应出现在公开文案中的内容

- AI、Codex、ChatGPT 或其他工具的对话记录、生成说明和提示词。
- TODO、占位文本、评审备注或“稍后补充”等草稿痕迹。
- 内部实现说明、接口细节、调试口吻和仅面向开发者的技术说明。
- 未核实的主体名称、地址、联系方式、备案号、授权关系或法律结论。
- 面向普通用户展示的调试账号、测试密码、Token、签名文件或本地环境信息。
- 与用户理解功能无关的框架模板链接和过多协议导航链接。

## README 写法

- README 可以面向开发和部署，但法律、隐私和安全提示应简洁明确。
- README 不应复制完整协议正文，也不应替代 App 内的正式协议和政策页面。
- README 不应发布真实或看起来真实的测试账号、密码、手机号、邮箱、Token 或签名信息。
- 如需说明公开文案维护规则，链接本指南即可。

## 协议日期处理

- 只修错别字、排版、语气或 README 说明时，不更新协议生效日期。
- 只有涉及用户权利义务、隐私处理、争议处理、校园凭证规则或服务规则的实质变更时，才更新协议日期。
- 更新日期时，应同步检查 Web、Android、iOS 和小程序端展示是否一致。

## 提交前检查

- 没有内部备注、AI 对话、TODO 或占位说明。
- 没有未核实的法律主体、联系方式、备案号或授权关系。
- 必要的隐私、校园凭证、权限、第三方服务和非官方服务提示仍然可见。
- README 没有重复大段协议正文，也没有变成协议目录页。
- 没有在无实质规则变化时更新协议日期。
- 没有加入本地构建产物、签名文件、证书文件或环境文件。
